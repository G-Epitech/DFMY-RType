# Server Usage
Welcome to the **R-Type server CLI** documentation. Each server type has its own CLI options.
This document provides detailed instructions on how to use the CLI for the **director** server.

## General Usage
The R-Type server CLI can be invoked with the following format:
```bash
./r-type_server [server_type] ...(type specific arguments)
```

### Available Server Types

- **Director**: The director server is responsible for managing the game sessions and routing messages between clients and game servers.
- **Lobby**: The game server is responsible for managing the game logic and state for a single game session.

Each server type has its own set of arguments and options, accessible via the ``--help`` flag.

## Director Server CLI

The ``director`` server is responsible for managing the infrastructure,
including spawning and scaling lobby servers. Below is the detailed usage guide for the director server.

### Usage
```bash
./r-type_server director [options]
```

### Available Options
| Option     | Argument   | Description                                 |
|------------|------------|---------------------------------------------|
| `--help`   | None       | Produces the help message for the director. |
| `--env`    | `<file>`   | Path to the director's environment file.    |

### Environment File
The ``--env`` option allows you to specify a configuration file that defines
key parameters for the director server. This environment file must follow the format outlined below.

#### Sample Environment File
```dotenv
NAME='jedusor'
PORT=3000
TICKS=60
MAX_GAMES=5
```

#### Environment Variables
| Variable    | Description                                                                  |
|-------------|------------------------------------------------------------------------------|
| `NAME`      | The name of the director server.                                             |
| `PORT`      | The port on which the director server runs.                                  |
| `TICKS`     | The tick rate (in milliseconds) for each game lobby managed by the director. |
| `MAX_GAMES` | The maximum number of games (lobby servers) that the director can spawn.     |

#### Example Usage
Here is an example of how you might use the **``director``** server:


```bash
./r-type_server director --env director.env
```

#### Further Help
For more information on the different server types and their specific CLI options, you can run:

```bash
./r-type_server [server_type] --help
```
